# Tame Risk Analysis

## Risks:
1. Technical Computer fails errors occur during the work on Project.
2. Website files and content are accidently deleted. 
3. GitHub Pages doesn't publish my pages correctly / new changes not shown on live site.
4. Broken navigation links between pages.
5. Missing the project due date.






## TAME Strategy How to Prepare for Each Risk Mentioned Above Using Risk Analysis

- **Transfer:** For risk # 1, portfolio website is stored in GitHub so that it can be accessed across devices such as another computer or my mobile phone. 

- **Accept:** For risk # 3, if issues occur with GitHub pages with my site not publishing, I will accept the issue if it occurs and troubleshoot the website by republishing it.

- **Mitigate:** For risks # 2 and 5, to prevent accidental deletion of files, I will commit changes and save regularly to GitHub and keep backups. To avoid missing the deadline I will follow a project schedule for each day to work on it and finish it as early as possible.

- **Eliminate:** For risk # 4, for broken links, I will test and fix every link before submitting the project.
