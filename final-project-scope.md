# **Project Scope Statement**

## Project Title: Personal Portfolio Website

## Project Purpose

The purpose of my project is to create and design a personal porfolio website that showcases everything about me such as skills, experience, education, contact info,and etc. This personal portfolio website will demonstrate my skills in website development using HTML and CSS and as well my skills in project management that I learned this semester. 

## Feedback Loops

The instructor of this course will provide feedback for this project as well as other assignments related to this project. Classmates will probably provide peer feedback on my portfolio site to suggest improvments, changes, etc before is final submission. Any feedback that I recieve from instructor and classmates will be incorporated into the final version of my personal portfolio site where necessary. 

## Description

My personal portfolio site will consist of two-page personal portfolio website involving the use of HTML and CSS code. The portfolio site will have a homepage that shows more about myself. It will have an additional page that will include my education, work experience, skills, and hobbies, as well as contact information. My two page site will use the same design format throughout the site and will be published using GitHub Pages.

The desired results of this portfolio site wiill be to provide a functioning two page portfolio website that is available for everyone to see on GitHub Pages. ## Priorities

1. Quality
2. Time
3. Budget

Quality is a high priority for me because the website needs to be professional, organized, and fully functional between pages tobe considered as good quality.

Time is a high priority because project must be completed by the deadline. Development of personal portfolio website will completed throughout the semester until it reaches the due date.

Budget is a low priority because the personal portfolio site uses free developer tools such as GitHub, Visual Studio Code, ChatGPT, and Google. No financial budget isn't necessary.

## Exclusions
The features below are not included or outside the scope of this project:

- Advanced Javascript coding
- 3D models or interactive 3D models
- User login accounts or systems
- Blogs
- Video streaming
- Complex/high tier animations

## Acceptance Criteria

- Project is accessible to everyone publicly using GitHub Pages.
- Navigation works properly between pages. No broken links between pages.
- HTML pages follow correct format and structure.
- The GitHub repository has all required documenation needed.
- No missing content.
- Correct spelling and grammar.
- Instructor approves completed personal portfolio site.
